# Admin Panel - Data Model

The Admin Panel is a client-side application and does not have its own database. Instead, it relies on the **Backend API** to fetch the data it needs to display. This document outlines the main data structures that the Admin Panel will consume from the API.

## Log

Represents a single error log submitted to the system.

-   `id` (string): A unique identifier for the log.
-   `status` (string): The current status of the log (e.g., "Pending", "In Progress", "Completed", "Failed").
-   `timestamp` (string): The time the log was submitted.
-   `content` (string): The full content of the error log.

## Fix

Represents a fix generated by the LLM for a specific error log.

-   `id` (string): A unique identifier for the fix.
-   `logId` (string): The ID of the log that this fix is for.
-   `timestamp` (string): The time the fix was generated.
-   `generatedFix` (string): The code or solution generated by the LLM.

## System Health

Represents the health status of a single microservice.

-   `serviceName` (string): The name of the service (e.g., "Backend API", "Worker Service").
-   `status` (string): The current status of the service (e.g., "Online", "Offline", "Degraded").
-   `lastChecked` (string): The last time the health of the service was checked.
