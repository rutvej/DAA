# System Health Feature - System Overview

The System Health feature provides a dedicated page in the Admin Panel for monitoring the real-time status of all the microservices that make up the application.

## Key Information

-   **Service Status:** A list of all services with their current status (e.g., "Online", "Offline", "Degraded").
-   **Last Checked:** The timestamp of the last health check for each service.
-   **Detailed Metrics:** (Optional) More detailed metrics for each service, such as response time and error rate.

## Purpose

The main purpose of the System Health page is to give administrators a centralized place to:

-   Quickly identify any services that are down or experiencing issues.
-   Monitor the overall stability and performance of the system.
-   Diagnose problems by seeing the status of all components in one place.
