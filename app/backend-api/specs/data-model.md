# Backend API - Data Model

This document outlines the data models for the main entities managed by the Backend API. These models define the structure of the data as it is stored in the **Database**.

## User

Represents a user account with access to the system.

-   `id` (string): A unique identifier for the user.
-   `username` (string): The user's chosen username.
-   `passwordHash` (string): A hashed version of the user's password for security.
-   `role` (string): The user's role (e.g., "Admin", "User").

## Log

Represents a single error log submitted to the system.

-   `id` (string): A unique identifier for the log.
-   `userId` (string): The ID of the user who submitted the log.
-   `status` (string): The current status of the log (e.g., "Pending", "In Progress", "Completed", "Failed").
-   `timestamp` (datetime): The time the log was submitted.
-   `content` (text): The full content of the error log.

## Fix

Represents a fix generated by the LLM for a specific error log.

-   `id` (string): A unique identifier for the fix.
-   `logId` (string): The ID of the log that this fix is for.
-   `timestamp` (datetime): The time the fix was generated.
-   `generatedFix` (text): The code or solution generated by the LLM.
-   `isApproved` (boolean): A flag to indicate whether the fix has been approved by an administrator.
